<INVENTION-TITLE>
Method of protecting security information.
</INVENTION-TITLE>
<CLASSIFICATIONS>G06F2100	G06F2100	</CLASSIFICATIONS>
<CLASSIFICATIONS-THIRD>G06F	G06F	</CLASSIFICATIONS-THIRD>
<CLASSIFICATIONS-FOURTH>G06F21	G06F21	</CLASSIFICATIONS-FOURTH>
<ABSTRACT>
Security information, such as cryptographic keys, are stored in the security module (SM) of a terminal. The terminal (T) is coupled to a central data processing system (DVA). An authentication takes place prior to a data exchange between the terminal (T) and the data processing system (DVA). In this authentication, the security module (SM) communicates a time interval, in an encrypted manner, to the data processing system (DVA), after the expiry of which interval a renewed authentication must be initiated by the data processing system (DVA). If this renewed authentication is not initiated in time, the security information contained in the security module (SM) is erased.
